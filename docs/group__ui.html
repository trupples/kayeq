Detailed Description

The ui module provides facilities for displaying terminal graphics to the user and for receiving input from the user.

The ui module "lifecycle" is as follows:

  1. ui_init()
  2. a series of drawing calls flushed by ui_to_screen() calls, or user input calls
  3. ui_reset()

All graphics assume an 25x80 terminal with ANSI escape sequence handling.

The ui_cursor() function introduces a dependency to the eq module.

◆ ui_to_screen()

void ui_to_screen ( )

Flushes everything that was drawn using ui_draw functions to the screen.

Until this function is called, the displayed graphics may not change. This is used instead of no buffering or line buffering to prevent flickering.
